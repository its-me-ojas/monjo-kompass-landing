# From MongoDB Compass to Monjo-Kompass: A Developer's Journey

Considering switching from MongoDB Compass to Monjo-Kompass? This guide covers everything you need to know.

## Why Make the Switch?

### The Problem with GUI Tools

MongoDB Compass is a great tool, but it has limitations:

**Resource Usage**:
- 300MB+ memory footprint
- 200MB+ disk space
- 2-5% CPU usage even when idle
- 3-5 second startup time

**Workflow Disruption**:
- Requires leaving the terminal
- Mouse-heavy interface
- Context switching breaks flow
- Slow for repetitive tasks

**Development Environment**:
- Doesn't integrate with terminal workflows
- Can't script operations easily
- Difficult to use over SSH
- Not ideal for remote servers

### The Monjo-Kompass Advantage

**Performance**:
- ~10MB memory footprint (30x smaller)
- 8MB binary size (25x smaller)
- &lt;1% CPU usage
- 50ms startup time (60x faster)

**Workflow**:
- Stay in your terminal
- Keyboard-driven interface
- Maintains flow state
- Fast for repetitive tasks

**Integration**:
- Works perfectly over SSH
- Scriptable operations
- Integrates with tmux/screen
- Clipboard integration

## Migration Guide

### Step 1: Installation

**Remove Compass** (optional):
```bash
# macOS
rm -rf /Applications/MongoDB\ Compass.app

# Windows
# Uninstall via Control Panel

# Linux
sudo apt remove mongodb-compass
```

**Install Monjo-Kompass**:
```bash
# From source
git clone https://github.com/its-me-ojas/monjo-kompass
cd monjo-kompass
cargo build --release
sudo cp target/release/kompass /usr/local/bin/

# Or download binary
# (Coming soon: cargo install kompass)
```

### Step 2: Import Connections

**Export from Compass**:
1. Open Compass
2. Go to Connections
3. Copy each connection string

**Import to Kompass**:
1. Open Monjo-Kompass
2. Paste connection string (Ctrl+V)
3. Press Enter to connect
4. Connection saved to history automatically

**Tip**: Keep a text file with your connection strings:
```bash
# connections.txt
mongodb://localhost:27017
mongodb://staging-server:27017
mongodb+srv://user:pass@prod-cluster.mongodb.net/
```

### Step 3: Learn the Basics

**Day 1**: Navigation
- Practice j/k movement
- Learn Enter to select
- Use Backspace to go back

**Day 2**: Filtering
- Press 'f' to filter
- Try simple filters: `{"status": "active"}`
- Practice clearing filters with Esc

**Day 3**: Queries
- Press 'q' for query mode
- Start with simple finds
- Progress to aggregations

**Week 1**: You'll be faster than Compass

### Step 4: Adapt Your Workflows

**Compass Workflow**:
```
1. Click MongoDB Compass icon (wait 3-5s)
2. Click "Connect"
3. Click database
4. Click collection
5. Click "Filter"
6. Type filter in GUI
7. Click "Apply"
8. Click document to view
Total: ~30-45 seconds
```

**Kompass Workflow**:
```
1. Type 'kompass' (instant)
2. Press Enter (connect from history)
3. Press j j j (navigate to database)
4. Press Enter
5. Press j (navigate to collection)
6. Press Enter
7. Press f (filter)
8. Type filter
9. Press Enter
Total: ~10-15 seconds
```

**Time Saved**: 20-30 seconds per operation
**Daily Operations**: ~50 times
**Time Saved Daily**: 15-25 minutes

## When to Use Which Tool

### Use MongoDB Compass When:

1. **You're new to MongoDB**
   - Visual interface is easier to learn
   - Schema analysis helps understand data
   - Query builder teaches syntax

2. **You need visual schema analysis**
   - Compass shows field types and distributions
   - Helpful for understanding unfamiliar databases
   - Great for presentations

3. **You're building complex aggregations**
   - Visual pipeline builder is helpful
   - Can see intermediate results
   - Easier to debug stages

4. **You prefer GUI tools**
   - Some people work better with visual interfaces
   - No judgment - use what works for you

### Use Monjo-Kompass When:

1. **You live in the terminal**
   - Vim/Emacs user
   - SSH into servers frequently
   - Use tmux/screen

2. **Performance matters**
   - Working on laptop with limited resources
   - Need instant startup
   - Running many tools simultaneously

3. **You do repetitive tasks**
   - Checking production databases frequently
   - Running similar queries often
   - Need speed over visual aids

4. **You're a keyboard power user**
   - Prefer keyboard over mouse
   - Know MongoDB query syntax
   - Value efficiency

### Use Both When:

**Development Workflow**:
- Compass for initial schema exploration
- Kompass for daily operations
- Compass for complex aggregation building
- Kompass for quick checks and queries

**Team Environment**:
- Compass for demos and presentations
- Kompass for personal productivity
- Compass for onboarding new developers
- Kompass for experienced developers

## Transition Tips

### 1. Start Small

Don't switch completely on day one:
- Use Kompass for simple tasks first
- Keep Compass for complex operations
- Gradually increase Kompass usage

### 2. Practice Keybindings

Spend 15 minutes practicing:
- Navigation (j/k)
- Selection (Enter)
- Going back (Backspace)
- Filtering (f)

### 3. Keep a Cheat Sheet

Print or save this:
```
j/k     - Navigate
Enter   - Select
Backspace - Back
f       - Filter
q       - Query
e       - Export
r       - Refresh
Ctrl+V  - Paste
```

### 4. Use Connection History

First time: Type/paste URI
Every time after: Select from history
Saves tons of time

### 5. Learn One Feature at a Time

Week 1: Basic navigation
Week 2: Filtering
Week 3: Query mode
Week 4: Aggregations
Week 5: Export
