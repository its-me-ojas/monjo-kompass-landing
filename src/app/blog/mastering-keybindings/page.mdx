## Advanced Features

### Document Filtering

**Use Case**: Find specific documents quickly

**Workflow**:
```
1. In document view, press 'f'
2. Enter filter: {"status": "active"}
3. Press Enter
4. Only matching documents show
```

**Filter Examples**:
```json
// Find by field value
{"status": "active"}

// Find by ID
{"_id": ObjectId("507f1f77bcf86cd799439011")}

// Find with operators
{"age": {"$gte": 18}}

// Multiple conditions
{"status": "active", "role": "admin"}
```

**Clear Filter**: Press `Esc` or `Backspace`

### Custom Query Mode

**Use Case**: Execute complex MongoDB queries

**Workflow**:
```
1. Press 'q' in document view
2. Enter query: db.users.find({age: {$gte: 18}}).limit(10)
3. Press Enter
4. Results display in document view
```

**Query Examples**:
```javascript
// Find with projection
db.users.find({}, {name: 1, email: 1})

// Sort results
db.events.find().sort({timestamp: -1}).limit(100)

// Count documents
db.users.countDocuments({status: "active"})

// Distinct values
db.events.distinct("event_type")
```

### Aggregation Pipelines

**Use Case**: Complex data transformations

**Workflow**:
```
1. Press 'q' in document view
2. Enter aggregation:
   db.events.aggregate([
     {$match: {event: "purchase"}},
     {$group: {_id: "$user_id", total: {$sum: "$amount"}}}
   ])
3. Press Enter
4. Results display
```

**Pipeline Examples**:
```javascript
// Group and count
[
  {$group: {_id: "$status", count: {$sum: 1}}}
]

// Filter and sort
[
  {$match: {age: {$gte: 18}}},
  {$sort: {created_at: -1}},
  {$limit: 100}
]

// Complex aggregation
[
  {$match: {date: {$gte: ISODate("2024-01-01")}}},
  {$group: {
    _id: {$dateToString: {format: "%Y-%m-%d", date: "$date"}},
    revenue: {$sum: "$amount"},
    count: {$sum: 1}
  }},
  {$sort: {_id: -1}}
]
```

### Export Functionality

**Use Case**: Save query results for analysis

**Workflow**:
```
1. Navigate to documents or run a query
2. Press 'e'
3. Choose format (JSON/CSV/BSON)
4. Enter filename
5. Press Enter
6. File saved to current directory
```

**Export Formats**:
- **JSON**: Human-readable, great for sharing
- **CSV**: Import into Excel/Google Sheets
- **BSON**: Native MongoDB format, preserves types

**Goal**: Export specific user records

```

1. Connect to database
2. Navigate to 'users' collection
3. Press 'f'

7. Choose JSON format
```

```
4. Press Enter

6. Press 'e' to export if needed
```

## Productivity Tips

### 1. Use Connection History

Never type the same URI twice:
- First connection: Type or paste URI
- Subsequent connections: Select from history
- Saves ~20 seconds per connection

### 2. Master Filtering

Instead of scrolling through thousands of documents:
```
Without filtering: 2-3 minutes to find document
With filtering: 5 seconds
```

### 3. Learn Query Mode

GUI query builders are slow. Command line is fast:
```
GUI: Click, select, type, click, wait... (30+ seconds)
Query mode: Type query, Enter (5 seconds)
```

### 4. Keyboard-Only Navigation

Never reach for the mouse:
```
Mouse workflow: 15-20 actions per minute
Keyboard workflow: 40-60 actions per minute
```

### 5. Use Vim Keybindings

If you use Vim/Neovim:
- Your muscle memory works instantly
- No context switching
- Stay in the flow

## Troubleshooting

### Connection Issues

**Problem**: "Connection failed"

**Solutions**:
1. Check URI format
2. Verify network connectivity
3. Check MongoDB server status
4. Verify credentials

### Filter Not Working

**Problem**: Filter returns no results

**Solutions**:
1. Check JSON syntax
2. Verify field names (case-sensitive)
3. Use `$regex` for partial matches
4. Clear filter with Esc and retry

### Query Syntax Error

**Problem**: "Query failed: syntax error"

**Solutions**:
1. Verify MongoDB query syntax
2. Check for missing commas/brackets
3. Test query in MongoDB shell first
4. Use simpler query and build up
